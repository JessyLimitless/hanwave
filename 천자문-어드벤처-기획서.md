## 1. 프로젝트 개요

| 항목 | 내용 |
|---|---|
| 게임명 | 천자문 어드벤처: 마법의 팝업 (Thousand Characters Adventure) |
| 장르 | 교육용 팝업 퀴즈 RPG (Edutainment Box-Quiz Game) |
| 타깃층 | 천자문을 쉽고 재미있게 독학하고자 하는 아동 및 성인 초보자 |
| 플랫폼 | Web (Responsive) → 추후 Mobile App (iOS / Android) |
| 핵심 컨셉 | '마법의 천자문' 감성의 모험 요소를 차용한 [스테이지 이동 → 팝업창 퀴즈 → 시각적 해설 및 카드 수집] 학습 루프 |

### 세계관
사용자는 봉인된 '마법의 천자문' 책을 발견하고, 봉인을 풀기 위해 한자 퀴즈를 해결해야 하는 모험가가 된다. 지도(Stage Map)를 이동하며 책을 한 장씩 넘기고, 새로운 책장마다 '한자 몬스터'가 갇혀 있어 팝업창 퀴즈로 이들을 물리친다.

---

## 2. 핵심 게임 루프 (Core Gameplay Loop)

```
[스테이지 지도 이동]
        ↓
[몬스터 조우 → 팝업창 생성]
        ↓
[4지선다 객관식 퀴즈]
        ↓
  ┌─────┴─────┐
정답              오답
  ↓                ↓
[어원 해설      [재도전 /
 & 카드 획득]    오답노트 이동]
```

### 단계 1 — 마법의 팝업창 등장
- 사용자가 지도에서 다음 스테이지를 클릭하면 화면 중앙에 애니메이션과 함께 화려한 팝업창(Pop-up Box)이 등장
- 팝업창 중앙에 강력한 에너지를 뿜어내는 한자(예: 天 - 하늘 천)가 크게 표시
- 하단에 4개의 객관식 보기 제시
- 한자 등장 시 '번쩍!' 시각 효과 + 음성 효과로 몰입감 부여

### 단계 2 — 객관식 맞추기 및 공격 (배틀)
- **정답 선택**: "성공!" 사운드 + 아바타가 몬스터를 공격하는 연출 → 팝업창 사라지고 몬스터 봉인
- **오답 선택**: "콰광!" 사운드 + 대미지 연출, 또는 다른 문제로 재도전

### 단계 3 — 한자 해설과 카드 획득 (가장 중요)
정답을 맞추면 같은 팝업창 내에서 해설 화면으로 전환:
- 한자의 정확한 뜻과 음
- 뜻 풀이: 왜 이런 모양과 뜻이 되었는지 흥미로운 어원 설명
- 예시 단어/성어 (실생활 예시)
- 학습 카드 형태로 자동 수집되어 도감에 저장

## 3. 게임 시스템 & UI 기획

### 3.1 월드 맵 (World Map)
- 총 1000자를 250개 관문(4자 1세트 × 250개 에피소드)으로 구성
- 챕터 단위 분류 예: 1장 - 기초 한자, 2장 - 자연, 3장 - 인간관계 등
- 고전 마법서가 펼쳐지는 듯한 황토색 양피지 스타일
- 현재 진행 중인 스테이지 아이콘 활성화, 이전 클리어 스테이지는 잠금 해제 표시

### 3.2 마법 팝업창 (Quiz Pop-up)
- 팝업 등장 시 책이 펼쳐지는 듯한 3D 효과 또는 마법진 효과
- 중앙: 이펙트가 감도는 대형 한자 박스
- 하단: 4개의 객관식 보기 버튼
- 한자 등장 시 강렬한 시각 이펙트 ('번쩍' 글로우 애니메이션)

### 3.3 해설 & 카드 (Result & Lore)
- 한자의 훈음(뜻/음), 부수, 총획 표시
- 그림 기반 어원 해설 + 추천 단어
- 정답 시 팝업창 내에서 탭이 전환되며 카드 도감에 자동 저장

### 3.4 천자문 카드 도감
- 수집한 한자 카드를 모아보는 컬렉션 화면
- 카드별 복습 기능: 쓰기 연습, 음성 듣기(TTS) 제공

## 4. 데이터 구조 (Schema)

```typescript
interface HanjaCharacter {
  id: string;              // 고유 식별자 (예: "ep01-01")
  episodeId: string;       // 소속 에피소드 (4자 1세트)
  character: string;       // 한자 표기 (예: "天")
  meaning: string;         // 뜻 (예: "하늘")
  sound: string;           // 음 (예: "천")
  radical?: string;        // 부수
  strokeCount?: number;    // 총획
  quizOptions: string[];   // 오답 포함 4개 보기 ["땅 지", "하늘 천", "검을 현", "누를 황"]
  answerIndex: number;     // 정답 인덱스
  originStory: string;     // 어원 중심 이야기식 해설
  vocabulary: { word: string; meaning: string }[]; // 활용 단어 예시
}

interface Episode {
  id: string;               // 예: "ep01"
  title: string;            // 예: "우주의 시작"
  characters: string[];     // 4개 한자 ID
  comboPhrase: string;      // 천지현황 등 사자성어
  comboTranslation: string; // "하늘은 검고 땅은 누르다"
}

interface PlayerProgress {
  userId: string;
  collectedCards: string[];     // 획득한 한자 ID 목록
  wrongNotes: string[];         // 오답 노트 (재도전 필요)
  currentStage: number;
  rank: string;                 // "초동달인" → "문장가" → "대제학"
  exp: number;
  gold: number;
}
```

### 데이터 샘플: 에피소드 1 — 우주의 시작 (天地玄黃)

| 한자 | 뜻/음 | 보기 구성 | 어원 해설 |
|---|---|---|---|
| 天 | 하늘 천 | ① 땅 지 ② 하늘 천 ③ 검을 현 ④ 누를 황 | 사람이 팔다리를 벌리고 선 모양 위에, '가장 높은 곳'을 뜻하는 선을 그어 하늘을 나타냄 |
| 地 | 땅 지 | ① 땅 지 ② 하늘 천 ③ 검을 현 ④ 누를 황 | 누에가 알을 낳는 모양(也)과 흙(土)이 합쳐져, 만물을 길러내는 땅을 의미 |
| 玄 | 검을 현 | ① 땅 지 ② 하늘 천 ③ 검을 현 ④ 누를 황 | 실타래를 묶은 모양에서 유래, 깊고 아득하여 눈에 잘 보이지 않는 검고 오묘한 빛 |
| 黃 | 누를 황 | ① 땅 지 ② 하늘 천 ③ 검을 현 ④ 누를 황 | 사람이 허리춤에 황금빛 옥(玉)을 차고 있는 모습에서, 풍요로운 흙과 황금의 색 |

## 5. 게이미피케이션 시스템

### 5.1 사자성어 완성 보너스 (Combo)
- 한 에피소드(4자)를 연속으로 정답 맞추면 전체 문장 해석 팝업 등장
- 예: "천지현황(天地玄黃): 하늘은 검고 땅은 누르다!" + 특별 보상(골드, 경험치)

### 5.2 오답 노트 (마법의 오답서)
- 틀린 한자는 자동으로 '틀린 마물' 저장소로 이동
- 복습 퀴즈를 다시 맞춰야 완전히 정화되어 카드 도감으로 복귀

### 5.3 성장 및 등급
- 카드 수집 개수에 따라 마법 등급 상승: **초동달인 → 문장가 → 대제학**

### 5.4 확장 아이디어 — 소환 시스템 (향후 버전)
- 수집한 한자 카드에 마력 부여 → 배틀 중 위급 시 '불 화(火)' 소환해 공격, '물 수(水)' 소환해 체력 회복

## 6. 기술 스택 제안 (Claude Code 구현 기준)

| 영역 | 제안 스택 | 비고 |
|---|---|---|
| 프레임워크 | Next.js 14 (App Router) | SocialBrain/MuseAI 기존 프로젝트와 톤 일치 |
| 언어 | TypeScript | |
| 스타일링 | Tailwind CSS | |
| 상태 관리 | Zustand 또는 React Context | 진행도, 카드 도감 등 클라이언트 상태 |
| 데이터베이스 | Supabase (Postgres) | 유저 진행도, 카드 컬렉션 저장 |
| 인증 | Supabase Auth | 게스트 모드 우선 고려 (간편 시작) |
| 애니메이션 | Framer Motion | 팝업 등장, 카드 플립 등 연출 |
| 음성(TTS) | Web Speech API 또는 외부 TTS API | 한자 발음 듣기 기능 |
| 1차 콘텐츠 데이터 | JSON 정적 파일 (100자) | MVP는 DB 없이 시작 가능 |

### MVP 범위 제안 (1차 구현)
1. 기초 100자 (25개 에피소드) 콘텐츠 JSON
2. 월드맵 → 팝업 퀴즈 → 해설 카드 핵심 루프
3. 로컬 상태 기반 카드 도감 (브라우저 저장, DB 없이)
4. 사자성어 콤보 보너스 연출
5. 오답 노트 기본 동작

> 프로토타입 HTML/CSS/JS 예시는 이미 보유 중이므로, Claude Code 세션에서는 이를 Next.js 컴포넌트 구조로 재구성하는 작업부터 시작하면 됨 (`QuizPopup`, `WorldMap`, `CardCollection`, `ComboReveal` 컴포넌트 분리 제안).

## 8. Claude Code 작업 시 참고 메모
- 한자 데이터(특히 어원 해설)는 분량이 많으므로, 콘텐츠 JSON은 별도 파일로 분리해 단계적으로 채워나가는 방식 권장 (에피소드 단위 커밋)
- 정답/오답 판정, 콤보 트리거, 카드 저장 로직은 핵심 로직이므로 최소 동작 버전(러너블)을 먼저 구현 후 연출 추가하는 순서 권장
- 팝업 애니메이션은 과한 스캐폴딩보다 Framer Motion 기본 트랜지션으로 먼저 동작시키고 점진적으로 디테일 추가
